Перелік основних команд:

№1

  • git config --global user.name blabla - вказати ім'я.
  • git config --global user.email blabal@bla - вказати email.
  • git init - створює новий пустий репозиторій.
  • git status - основний інструмент, який використовується для визначення, які файли в якому стані знаходяться.
  • git add - для того щоб почати відстежувати новий файл.
  • git commit - це команда для запису індексованих змін в репозиторій.
  • git log - виводить список коммітов створених в даному репозиторії в зворотному хронологічному порядку.
  • git checkout - переключення між двома вітками, віновення старої версії.
  • git branch – це команда для управління гілками в репозиторії Git.
  • git push - вливання локальних змін в віддалений репозиторій.
  • git pull - отримати дані з віддаленого репозиторію і обєднати із своїми.
  • git merge - вносить зміни з віддаленого репозиторію на локальний.
  • git clone - завантажує репозиторій.

№2

.gitignore потрібен для приховування файлів і папок від системи контролю версій Git.

№3 Advanced

  • git revert - створює "антікомміт" для зазначеного комміта, таким чином скасовуює зміни, внесені в попередні.
  • git reset - відміняє зміни які ще не відправлені на віддалений репозиторій.
  • git rebase - об'єднання віток.
  • git cherry-pick - застосовує зміни, внесені деякими існуючими коммітами.
  • git stash - використовується для того щоб, зберегти свої зміни (а потім, коли потрібно, повернути їх).
